AWS EC2 + Oracle Database Deployment & Security

Project Overview

In this project, I successfully deployed and configured an Oracle database on an AWS EC2 instance running Windows Server 2022. The objective was to build a secure, production-ready database environment in the cloud, implementing proper security configurations including firewall rules, network access controls, and connection validation. This hands-on deployment demonstrated my proficiency with AWS cloud infrastructure, Windows server administration, and Oracle database setup.

Environment Setup

AWS EC2 Instance Provisioning

I began by launching an EC2 instance through the AWS Management Console. I configured the instance with the following specifications:

  • Operating System: Windows Server 2022 Base (Free Tier eligible)
  • AMI Selection: Microsoft Windows Server 2022 Base from the quick-start options

Initial Configuration Screen

Key Pair Creation & SSH Access

To secure remote access to the instance, I created a dedicated RSA key pair for authentication:

  • Key Pair Name: Key_[Username] (following organizational naming conventions)
  • File Format: Downloaded the private key for Remote Desktop Protocol access
  • Storage: Securely stored the key for future connections

RSA Key Pair Generation & Download

Phase 1: Remote Desktop Connection & Instance Access

Establishing Remote Desktop Session

I utilized the Remote Desktop Protocol (RDP) to establish a secure connection to the Windows Server instance:

  • Authentication: Administrator account with the instance password from EC2 console
  • Connection Method: Downloaded .rdp file executed locally
  • Network Setup: Connected through the AWS-provided security group

Remote Desktop Session to Windows Server Instance

System Configuration & Command Execution

After establishing the remote session, I executed critical system configuration commands on the Windows Server instance to prepare the environment for Oracle database installation. These commands configured network settings, system parameters, and prerequisite services.

Command Execution Results on Windows Server

Phase 2: Network Security Configuration

Security Group Configuration

I configured the AWS EC2 security group to enable database connectivity by modifying inbound traffic rules:

  • Access Level: Configured Oracle database port (1521) for access from any IP address
  • Rule Type: Oracle-RDS protocol specification
  • Security Group: Applied to the launch-wizard-1 security group

Security Group Inbound Rules Configuration Interface

Inbound Rule Implementation

I created a new inbound rule with appropriate naming conventions to document the database access configuration:

  • Rule Description: "[Full Name] Oracle SQL 1521 Inbound"
  • Port Range: 1521 (Oracle listener port)
  • Source: 0.0.0.0/0 (accessible from all IPs)

Oracle Database Inbound Rule Added to Security Group
