Symmetric-Key Ciphers Analysis with CrypTool-1

Project Overview

In this project, I analyzed and broke multiple symmetric-key ciphers using CrypTool-1, a cryptanalysis platform. I worked with both known and unknown plaintext scenarios, applying various cryptographic techniques including frequency analysis, known plaintext attacks, and brute-force methodology. The project deepened my understanding of classical cipher mechanisms and modern encryption vulnerabilities.

Environment Setup

I utilized CrypTool-1 as my primary analysis platform. This tool provides a comprehensive suite of cryptanalysis features including: - Symmetric cipher encryption/decryption - Frequency analysis and N-gram tools - Known plaintext attack capabilities - Manual substitution analysis - Brute-force key estimation

Playfair Cipher Identification

Cipher 6 exhibited a characteristic of Playfair encryption—an extra character beyond the original plaintext length. This occurs when the plaintext contains duplicate letters in a pair. In the word "TREES," the pair "EE" (at positions 79-80) requires a filler character, confirming this as a Playfair cipher.

Phase 4: Brute-Force Attack Analysis

DES Encryption with Partial Known Key

I encrypted the plaintext "I am laxman puri" using DES (ECB) with the key:

Key: 00 00 00 00 00 ff ff ff

DES encrypted output

I then performed a brute-force attack, constraining the first 10 hexadecimal digits to zeros. CrypTool performed a 21-bit brute-force search with estimated key recovery in approximately 14 seconds:

DES (ECB) brute-force analysis — 21-bit search, ~14 seconds remaining

Increasing Constraint Difficulty

I modified the key to 00 00 00 00 0f ff ff ff and repeated the attack. CrypTool performed a 24-bit brute-force search with estimated time of approximately 2 minutes:

DES (ECB) brute-force analysis — 24-bit search, ~1:54 remaining

Scaling the Search Space

With the key 00 00 00 00 ff ff ff ff, providing one fewer hexadecimal digit to constrain, the estimated attack time jumped to approximately 29 minutes—demonstrating the exponential growth of the search space.

DES (ECB) brute-force analysis — 28-bit search, ~29 minutes remaining

Each additional unknown bit exponentially increases the search time, clearly demonstrating how key length provides security against brute-force attacks.

Key Takeaways

  1. Frequency Analysis Effectiveness: Classical ciphers like Caesar, Affine, and Vigenere remain vulnerable to frequency analysis and known plaintext attacks—demonstrating why modern algorithms are essential.

  2. Cipher Identification: Each cipher type leaves distinctive fingerprints—frequency patterns for substitution, matching frequencies for transposition, and length anomalies for Playfair.

  3. Avalanche Effect: Modern block ciphers like AES exhibit strong avalanche properties, where minimal key changes produce drastically different ciphertexts, enhancing security.

  4. Exponential Key Space Growth: Each additional unknown bit approximately doubles the brute-force attack time, underscoring the exponential nature of cryptographic key spaces.

  5. Algorithm Comparison: Across tested algorithms, attack times scale from seconds to days with similar constraints, reflecting differences in cipher complexity and key schedules.
